# Implementation Plan: Code Quality Fixes

**Branch**: `002-fix-code-quality` | **Date**: 2026-06-13 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `/specs/002-fix-code-quality/spec.md`

## Summary

Address all code quality issues identified in REVIEW.md: eliminate silent error swallowing (`.ok()`) by routing errors to user notifications or logs, remove all `unwrap()`/`expect()` violations in non-test code, decompose workspace.rs from 1171 to under 800 lines, centralize magic numbers into an `AppConfig` struct, and add missing LICENSE and CHANGELOG files.

**Testing**: `cargo test` (33 unit tests in utils/encoding/recent_files); manual integration via `test-step.md`

**Project Type**: Desktop application (single binary, single window)

**Performance Goals**: No regressions — refactoring only, no new hot paths

**Constraints**: Zero panics in production (Constitution Principle II), no new source files (Principle IV)

**Scale/Scope**: 12 source files, ~4300 lines, 19 `.ok()` sites, 3 `unwrap()`/`expect()` violations

## Implementation Phases

### Phase A: LICENSE + CHANGELOG (FR-005, FR-006)

**Files**: LICENSE (new), CHANGELOG.md (new)
**Risk**: None — additive, no code changes

1. Download GPL-3.0-or-later text from GNU and write to LICENSE
2. Generate CHANGELOG.md from `git log --oneline`, grouped by commit purpose

### Phase B: Centralize Magic Numbers (FR-008)

**Files**: `src/utils.rs`, `src/workspace.rs`, `src/recent_files.rs`
**Risk**: Low — constants don't change behavior

1. Add `AppConfig` struct to `utils.rs` with all tunable constants
2. Replace hardcoded literals in workspace.rs with `AppConfig::FIELD` references
3. Replace `20` max recent files in recent_files.rs with `AppConfig::MAX_RECENT_FILES`

### Phase C: Eliminate `unwrap()`/`expect()` (FR-004)

**Files**: `src/main.rs`, `src/workspace.rs`
**Risk**: Low — straightforward replacements

1. `main.rs:137` — Replace `expect()` with `?` and error print
2. `main.rs:402` — Replace `expect()` with `?` and error print
3. `workspace.rs:1143` — Replace `unwrap()` with `if let` pattern

### Phase D: Fix Silent Error Swallowing (FR-003, FR-009, FR-010)

**Files**: `src/workspace.rs`, `src/main.rs`, `src/command_center.rs`, `src/file_watcher.rs`, `src/recent_files.rs`, `src/search.rs`, `src/topbar.rs`
**Risk**: Medium — most widespread change, 19 call sites

1. Classify each `.ok()` site using the three-tier contract (see contracts/error-handling.md)
2. Replace user-facing `.ok()` with `show_notification()` (workspace.rs:495,497,1166; command_center.rs:166; search.rs:233)
3. Replace background `.ok()` with `.log_err()` (file_watcher.rs:37,38,75,76; recent_files.rs:39,40; workspace.rs:105,223,884; topbar.rs:173)
4. Replace IPC `.ok()` with `eprintln!()` + early return (main.rs:253,345,384,411)

### Phase E: Decompose workspace.rs (FR-007)

**Files**: `src/workspace.rs`, `src/search.rs`, `src/diff_view.rs`
**Risk**: Medium — logic extraction, must maintain all behavior

1. Move search find/replace navigation logic (~200 lines) from workspace.rs to search.rs
2. Move file comparison logic (~50 lines) from workspace.rs to diff_view.rs
3. Verify workspace.rs < 800 lines
4. Run `cargo build` and `cargo test` to confirm no regressions

### Phase F: Validation
